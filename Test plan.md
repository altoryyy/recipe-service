<a name="introduction"/>

## 1. Введение

Данный документ описывает план тестирования веб-приложения [RecipeService](https://github.com/altoryyy/recipe-service).  
Документ предназначен для участников команды разработки и тестирования.  
Цель тестирования — проверить, соответствует ли поведение приложения требованиям, описанным в [SRS](https://github.com/altoryyy/recipe-service/blob/main/Requirements/SRS.md).

---

<a name="items"/>

## 2. Объект тестирования

К объектам тестирования относятся основные функции веб-приложения RecipeService:

* Добавление и просмотр рецептов;  
* Пошаговое выполнение рецептов;  
* Добавление, редактирование и удаление отзывов;  
* Поиск рецептов по ингредиентам;  
* Сортировка рецептов по кухне разных стран;  
* Отображение списка рецептов.

---

<a name="quality"/>

## 3. Атрибуты качества

1. **Функциональность:**  
   - Все заявленные функции должны выполняться корректно;  
   - Все формы должны проходить валидацию перед отправкой данных.  

2. **Удобство использования:**  
   - Интерфейс должен быть понятен и интуитивен;  
   - Пошаговый режим должен корректно отображаться на мобильных устройствах;  
   - Все кнопки и поля имеют осмысленные подписи.  

3. **Производительность:**  
   - Время загрузки страниц не должно превышать 2 секунд.  

4. **Безопасность:**  
   - Администрирование и модерация доступны только авторизованным пользователям (при наличии);  
   - Данные рецептов и отзывов сохраняются в базе данных.

---

<a name="risk"/>

## 4. Риски

Возможные риски при тестировании и эксплуатации:

* Отсутствие соединения с сервером API;  
* Ошибки фильтрации и сортировки;  
* Потеря данных при незавершённом заполнении формы;  
* Некорректная работа интерфейса на мобильных устройствах;  
* Несовместимость с отдельными версиями браузеров.

---

<a name="features"/>

## 5. Аспекты тестирования

Планируется тестирование следующих аспектов приложения:

### Добавление и просмотр рецептов
- Проверка формы добавления рецепта;  
- Проверка обязательных полей;  
- Просмотр рецептов в общем списке.

### Пошаговое выполнение рецепта
- Переход в пошаговый режим;  
- Корректность переключения между шагами;  
- Адаптация интерфейса под разные устройства.

### Работа с отзывами
- Добавление нового отзыва;  
- Редактирование отзыва;  
- Удаление отзыва;  
- Отображение средней оценки рецепта.

### Поиск и фильтрация
- Поиск по одному и нескольким ингредиентам;  
- Проверка корректности фильтрации списка рецептов;  
- Сортировка по кухням разных стран.

### Редактирование и удаление рецептов
- Проверка редактирования рецепта;  
- Удаление рецепта и обновление списка.

---

<a name="approach"/>

## 6. Подходы к тестированию

Используется **ручное функциональное тестирование**.  
Проверка проводится в браузерах **Google Chrome** и **Mozilla Firefox**.  
Инструменты:  
- React DevTools;  
- Вкладки Console и Network в DevTools для проверки запросов API.  

---

<a name="pass"/>

## 7. Представление результатов

Результаты тестирования оформлены в отдельном документе [Test Results](Test%20Results.md).  
Каждый тестовый сценарий содержит следующие поля:
| ID | Сценарий | Действие | Ожидаемый результат | Фактический результат | Статус |

<a name="conclusion"/>

## 8. Выводы

Данный тестовый план охватывает все основные функции приложения RecipeService.  
Успешное прохождение тестов подтверждает корректность работы системы в штатных условиях.  
Возможные дополнительные виды тестирования (нагрузочные, кроссбраузерные, UX-тесты) могут быть добавлены на следующих этапах разработки.
